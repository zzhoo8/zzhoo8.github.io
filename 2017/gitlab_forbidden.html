解决Gitlab forbidden问题

解决Gitlab forbidden问题

>

问题

从任何外部主机访问Gitlab都返回forbidden

[root@iZ94q9yb0qaZ nginx]# curl -v 172.18.0.8:10080
* About to connect() to 172.18.0.8 port 10080
*   Trying 172.18.0.8...
* Connected to 172.18.0.8 (172.18.0.8) port 10080
> GET / HTTP/1.1
> User-Agent: curl/7.29.0
> Host: 172.18.0.8:10080
> Accept: */*
>
< HTTP/1.1 403 Forbidden
< Server: nginx
< Date: Thu, 20 Jul 2017 09:01:30 GMT
< Content-Type: text/plain
< Content-Length: 10
< Connection: keep-alive
< Cache-Control: no-cache
< X-Request-Id: 3400e1a0-08d7-4d6b-95ee-ab2ab927dd0c
< X-Runtime: 0.000882
<
Forbidden
* Connection 0 to host 172.18.0.8 left intact

解决

进入Gitlab的container中，输入

root@repo:/# /opt/gitlab/embedded/bin/redis-cli -s /var/opt/gitlab/redis/redis.socket --scan | grep rack::attack
cache:gitlab:rack::attack:allow2ban:ban:172.18.0.2

删除rack机制产生的key即可

redis /var/opt/gitlab/redis/redis.socket> del cache:gitlab:rack::attack:allow2ban:ban:172.18.0.2
(integer) 1

Bingo
